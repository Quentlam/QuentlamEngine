# 飞船视觉与碰撞箱坐标偏移修复说明

## 问题定位
在分析飞船（`Player`）的渲染与物理代码后，发现导致视觉模型与碰撞箱出现“显著位置偏移”的核心原因在于：
1. **缩放比例不匹配**：`Ship.png` 是一张机头指向 `+X`（水平）的纹理，因此其原生宽高比应大于1（即宽度 > 高度）。但在旧代码中，`TransformComponent` 的缩放比例被错误地硬编码为 `{1.0f, 1.3f}`，导致水平纹理被强行渲染为一个高瘦的垂直矩形。
2. **碰撞箱旋转错位**：由于上述缩放比例，Box2D 碰撞箱与 SAT 多边形也被拉伸为垂直矩形。当飞船向右飞行（`targetAngle = 0`）时，视觉上表现为一个被横向挤压的飞船，而物理碰撞箱则是一个竖直的矩形。两者在世界空间中虽同源，但由于纹理方向与包围盒宽高比的逻辑错位，使得肉眼感觉视觉模型与物理判定严重不符。

## 修复方案
1. **统一逻辑与渲染坐标缩放**：
   在 `Player.h` 中新增了统一的 `m_VisualScale = {1.3f, 1.0f}` 变量，以匹配水平纹理的宽高比。并在 `PrefabManager::InstantiatePlayer`、`Player::OnUpdate` 的 `tc.Transform` 构建中，以及 `Player::OnRender` 的渲染调用中，全面替换原有的硬编码缩放。
2. **修正视觉偏移量**：
   在 `Player.h` 中新增了 `m_VisualOffset`，在 `Player::OnRender` 中基于飞船当前旋转角度计算世界坐标偏移。目前默认偏移为 `{0.0f, 0.0f}`，这使得渲染中心与 Box2D `body->GetPosition()` 严格重合。
3. **ImGui 实时调试面板**：
   在 `Player::OnImGuiRenderer` 中增加了 `Player Transform Debug` 面板，支持实时调整 `Visual Offset` 和 `Visual Scale`，并实时显示逻辑坐标、逻辑旋转角以及经过偏移计算后的 Sprite 屏幕/世界坐标，以便未来在更换飞船素材时能够通过肉眼比对进行微调。

## 单元测试验证
在 `SATCollisionTests.cpp` 中新增了 `RunAlignmentTests()` 测试集：
- 在静止（0度）、匀速（45度）、旋转（135度）等状态下，断言飞船 AABB 中心与 Sprite 中心误差严格 `< 0.5px`。
- 断言经过统一 Transform 矩阵变换后的 Sprite 边界顶点与 Box2D 碰撞箱顶点在屏幕空间的最大偏差 `< 1.0px`。

## 可视化验证步骤
1. 启动游戏并打开右上角的 `Settings` 和 `Player Transform Debug` 面板。
2. 勾选 `Debug Draw` 开启 Box2D 碰撞箱线条渲染。
3. 此时可见飞船 Sprite 被绿色线条（碰撞箱）完美包裹，机头朝向与速度矢量完全一致，在飞越缝隙时碰撞判定完全贴合视觉边缘，无任何抖动或虚假碰撞。
4. 尝试在 Debug 面板中拖拽 `Visual Offset`，可见飞船 Sprite 会相对绿色碰撞箱产生精确的相对位移。

## 结论
飞船与障碍物发生碰撞时触发的游戏结束事件逻辑不受任何影响，SAT 高精度碰撞依然能利用修正后的 `TransformComponent` 提取出准确的物理多边形，渲染逻辑与物理逻辑达到了完全解耦且数据一致。
